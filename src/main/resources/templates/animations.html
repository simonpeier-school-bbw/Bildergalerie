Animations

Transitions

CSS transitions allows to change property values smoothly, over a given duration. It is mostly used in combination with the :hover class.

Property Description
transition A shorthand property for combining all transition properties on one line
transition-delay Specifies a delay for the start of a transition
transition-duration Specifies how long a transition takes to complete
transition-property Specifies the CSS property to change
transition-timing-function Specifies in which time one animation cycle should be completed

Example

This transition changes the size, color and the border radius on hover. It uses the speed curve function ease-in-out.

@keyframe

The css style defined inside a keyframe rule, will gradually change the current style to the new one at certain times.

Property Description
animation A shorthand property for combining all animation properties on one line
animation-delay Specifies a delay for the start of an animation
animation-direction Specifies in which direction an animation should be played (forwards, backwards, alternate cycles)
animation-duration Specifies in which time one animation cycle should be completed
animation-fill-mode Specifies a style for the element when the animation is not playing (before it starts, after it ends, or both)
animation-iteration-count Specifies the number of times an animation should be played
animation-name Specifies the name of the @keyframes animation
animation-play-state Specifies whether the animation is running or paused
animation-timing-function Specifies the speed curve of the animation

Example

This animation uses the keyframe rule used to animate a square div. It gradually changes the color and changes the position. Furthermore the border-radius changes throughout the animation. It repeats itself for an infinite amount of time. CSS only!!!

Canvas animations

With Canvas animations we can create interactive animations. Unlike the keyframes and transitions, canvas animations are created in Javascript. The biggest disadvantage is, that once a shape is drawn, we cannot change it. If we want to move it, we have to redraw it. If the animations are complex, it depends on the performance of the computer, how fast it will load.

Example

This canvas animation is copied from Mozilla. It shows a super simple basic animation of the earth in our solar system. It makes use of the "rotate" method.

SVG animations

SVG animations are created directly in html with the svg tag. To animate them we use the animate tag. With svg animations we can create simple but also complex animations. The only problem with complex svg's is, that the code gets long and confusing.

Examples

This example creates a rectangle and changes its opacity from 1 to 0 in 5s and repeats itself indefinitely.

Here we create two circles. They are animated exactly the same, but animation of the blue circle is started 0.01s later than the orange one. That's why the orange circle has kind of a blue "shadow". Click on the circle to animate it!

This example shows a bouncing ball. It uses the "values", "keyTimes" and "keySplines" attribute. With the values we define the path the ball will follow. The "keyTimes" define in which time the circle should reach the values. With the keySplines we define the speedcurve (bezier curve) the "keyTimes" follow, so that we get this bouncing effect. Click on the circle to animate it!
